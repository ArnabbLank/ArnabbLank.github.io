The Problem

Managing environment variables across local dev, staging, and production is painful. Most Python apps use python-dotenv, but it lacks type safety, validation, and multi-source support.

What I Built

  • Type-safe env var loading with Pydantic model validation
  • Multi-source backends: .env files, AWS SSM, HashiCorp Vault
  • Built-in CLI for generating .env templates from code
  • Pluggable architecture — add custom backends in <20 lines
  • Auto-detection of missing/invalid variables at startup

Key Decisions

Pydantic over attrs: Pydantic's validator ecosystem and JSON schema generation made it the natural choice for env validation.

Click for CLI: Provides auto-generated help text and composable command groups with minimal boilerplate.

The Problem

Predicting when a turbofan engine will fail is critical for maintenance scheduling. Traditional methods lack uncertainty estimates, making them unreliable for safety-critical decisions.

Approach

  • CNN-LSTM hybrid architecture for temporal feature extraction
  • Transformer attention for long-range sensor dependencies
  • Monte Carlo dropout for uncertainty quantification
  • RAG-powered copilot that explains failure modes in natural language
  • Automated root cause analysis with LangChain agent

Results

The model achieves 92% accuracy on the NASA C-MAPSS benchmark while providing calibrated uncertainty bounds. The agentic copilot can trace sensor anomalies back to specific component degradation patterns.

The Problem

AI coding assistants like Claude and Cursor can't natively access GitHub repositories. You have to manually copy-paste code or describe your repo structure in natural language.

What I Built

  • MCP-compliant server using the FastMCP framework
  • Tools for listing repos, reading files, searching code
  • Async HTTP via httpx for fast GitHub API calls
  • Zero-config setup — just add your GitHub token
  • Works with Claude Desktop, Cursor, and any MCP client

Design Philosophy

Minimal surface area: Only expose the tools that AI assistants actually need. No bloat, no unnecessary abstractions. The entire server is ~140 lines because it does exactly one thing well.
